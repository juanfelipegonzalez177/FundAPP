# PRODUCT BACKLOG
## Sistema de Información para el manejo de voluntarios, actividades ambientales, donaciones, en la fundación Biosferas
### FUNDAPP X BIOSFERAS — 2025

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

| Código HU | Código CU | Historia de Usuario | Criterios de Aceptación | Escenarios | Prioridad |
|-----------|-----------|---------------------|------------------------|------------|-----------|
| **US001** | CU001 | COMO usuario NECESITO registrarme en la plataforma PARA poder tener un perfil personal. | - El correo electrónico debe ser único.<br>- La contraseña debe tener al menos 8 caracteres, una mayúscula, una minúscula, un número y un carácter especial.<br>- El sistema debe validar el correo electrónico verificando que existe.<br>- Todos los campos obligatorios (nombre, apellidos, fecha de nacimiento, correo, contraseña) deben ser llenados. | **Positivos:**<br>- El usuario ingresa toda la información correctamente (correo, nombres, apellidos, fecha de nacimiento, contraseña) y el sistema crea la cuenta con éxito.<br>- El sistema envía un correo de confirmación y el usuario puede verificar su cuenta sin problemas.<br><br>**Negativos:**<br>- El correo electrónico ya está registrado en el sistema. El sistema muestra un mensaje de error.<br>- La contraseña no cumple con los requisitos mínimos de seguridad.<br>- El correo de confirmación nunca llega, por lo que el usuario no puede activar su cuenta. | Alta |
| **US002** | CU002 | COMO usuario NECESITO iniciar sesión PARA acceder a mi perfil. | - El sistema debe validar el correo electrónico y la contraseña ingresados.<br>- Permitir la opción de recuperación de contraseña en caso de olvido. | **Positivos:**<br>- El usuario ingresa su correo y contraseña correctos y accede a su perfil sin inconvenientes.<br><br>**Negativos:**<br>- El usuario ingresa una contraseña incorrecta. El sistema muestra un mensaje de error como "Contraseña incorrecta".<br>- El correo electrónico no está registrado en el sistema. El sistema muestra un mensaje de error.<br>- El usuario ha intentado iniciar sesión demasiadas veces con datos incorrectos. La cuenta se bloquea temporalmente. | Alta |
| **US003** | CU003 | COMO usuario, NECESITO recuperar mi contraseña y actualizar mis datos personales PARA mantener mi información al día. | - El usuario puede recuperar la contraseña olvidada mediante un código enviado a su correo electrónico.<br>- Se puede acceder a una sección de perfil para editar los datos personales.<br>- Los datos que se pueden editar son: nombre, correo, teléfono, dirección, fecha de nacimiento e identificación.<br>- El sistema debe guardar los cambios realizados. | **Positivos:**<br>- El usuario solicita recuperar la contraseña, recibe el código en su correo electrónico y lo ingresa correctamente para restablecerla.<br>- El usuario accede a su perfil, edita su número de teléfono y la dirección, y los cambios se guardan exitosamente.<br><br>**Negativos:**<br>- El usuario ingresa un correo electrónico que no está registrado en el sistema.<br>- El código de recuperación ha expirado, y el usuario no puede usarlo.<br>- El usuario ingresa el código de recuperación de manera incorrecta.<br>- El usuario intenta guardar la información, pero hay un problema técnico y los cambios no se aplican.<br>- El usuario intenta actualizar un campo que no está permitido (por ejemplo, el correo principal de la cuenta) y el sistema no lo permite. | Baja |
| **US004** | CU004 | COMO usuario registrado necesito poder mirar las actividades disponibles PARA así poder escoger. | - El usuario puede ver una lista de las actividades disponibles.<br>- Se puede llenar un formulario de postulación que incluya nombres y apellidos, correo electrónico, fecha y la actividad a realizar.<br>- El sistema debe notificar al administrador sobre la nueva postulación. | **Positivos:**<br>- El usuario navega a la sección de postulación y ve una lista clara de todas las actividades que se ofrecen.<br><br>**Negativos:**<br>- No hay actividades disponibles para mostrar.<br>- Las actividades disponibles no tienen información clara (como la fecha, hora o descripción). | Media |
| **US005** | CU005 | COMO usuario registrado NECESITO postularme como voluntario PARA contribuir a la fundación. | - El usuario puede ver una lista de las actividades disponibles.<br>- Se puede llenar un formulario de postulación que incluya nombres y apellidos, correo electrónico, fecha y la actividad a realizar.<br>- El sistema debe notificar al administrador sobre la nueva postulación. | **Positivos:**<br>- El usuario completa el formulario de postulación y el sistema lo envía correctamente al administrador.<br><br>**Negativos:**<br>- El usuario deja campos obligatorios del formulario en blanco.<br>- El sistema no notifica al administrador sobre la nueva postulación, lo que causa un retraso.<br>- La actividad ya está llena y el sistema muestra un mensaje de error, impidiendo la postulación. | Media |
| **US006** | CU006 | COMO usuario NECESITO recibir una notificación para saber si salí seleccionado PARA la actividad a la que me postulé, también al momento en que se apruebe un certificado y la confirmación que si se realizó una donación o no. | - El usuario puede ver los resultados de la selección en la sección de su perfil.<br>- Se debe validar si la actividad aún está disponible; de lo contrario, el administrador la rechazará.<br>- Se enviará un correo de notificación para que el usuario sepa que se aprobó su participación, también al momento en el que se le apruebe un certificado, al igual que si realiza una donación exitosa. | **Positivos:**<br>- El usuario accede a su perfil y puede ver claramente si su postulación fue aceptada o rechazada.<br>- El correo electrónico de la notificación llega y el usuario puede rectificar y proceder.<br><br>**Negativos:**<br>- La información de selección no está disponible a tiempo o no es clara para el usuario.<br>- El administrador rechaza la postulación porque la actividad ya no está disponible, pero el sistema no notifica al usuario.<br>- La notificación nunca llega al correo y el usuario no puede saber si fueron aprobadas sus solicitudes.<br>- El usuario nunca se entera de que fue seleccionado y expira el tiempo. | Baja |
| **US007** | CU007 | COMO usuario registrado NECESITO realizar una donación PARA contribuir económicamente a la fundación. | - El usuario puede seleccionar un monto a donar.<br>- El usuario puede seleccionar un método de pago.<br>- Se deben solicitar los siguientes datos: nombres y apellidos, monto a donar, correo electrónico y métodos de pago. | **Positivos:**<br>- El usuario selecciona un monto, elige el método de pago, y la donación se procesa exitosamente.<br><br>**Negativos:**<br>- El método de pago falla o es rechazado.<br>- La información del usuario no se registra correctamente con la donación.<br>- El pago es rechazado debido a: fondos insuficientes, el método de pago no funciona. | Media |
| **US008** | CU008 | COMO usuario registrado NECESITO recibir un certificado digital PARA tener un reconocimiento oficial de mi ayuda. | - El sistema genera un certificado en formato PDF automáticamente cuando se cumple la ayuda y es aprobada.<br>- El certificado debe incluir el nombre y apellidos del usuario, horas, documento de identidad y el tipo de certificado. | **Positivos:**<br>- Una vez que el usuario cumple con su ayuda, el sistema genera automáticamente el certificado PDF y le permite descargarlo.<br><br>**Negativos:**<br>- El sistema no genera el certificado a tiempo o lo genera con información incorrecta.<br>- El usuario no puede descargar el archivo PDF. | Baja |
| **US009** | CU009 | COMO administrador NECESITO iniciar sesión PARA acceder a la información. | - Se puede ingresar con un usuario y contraseña específicos para administradores. | **Positivos:**<br>- El administrador ingresa sus credenciales exclusivas y accede al panel de control.<br><br>**Negativos:**<br>- Un usuario normal intenta usar la sección de inicio de sesión del administrador y es rechazado.<br>- El administrador olvida su contraseña. | Alta |
| **US010** | CU010 | COMO administrador NECESITO aprobar o rechazar las postulaciones de voluntarios PARA su participación. | - Se puede ver una lista de los voluntarios postulados.<br>- El administrador tiene la opción de aprobar o rechazar la postulación. | **Positivos:**<br>- El administrador puede ver una lista de todas las postulaciones y aprobar o rechazar cada una de ellas.<br><br>**Negativos:**<br>- La lista de postulaciones está desordenada o no muestra la información completa del voluntario.<br>- El administrador no puede rechazar una postulación, sólo aprobarla. | Media |
| **US011** | CU011 | COMO administrador NECESITO emitir los certificados de participación y de donación PARA llevar un control de los reconocimientos. | - El administrador puede generar un certificado para cualquier usuario.<br>- Se deben registrar los siguientes datos: nombre del voluntario, actividad realizada, fecha de la actividad y horas de voluntariado. | **Positivos:**<br>- El administrador puede generar un certificado rápidamente para cualquier usuario, ingresando los datos requeridos.<br><br>**Negativos:**<br>- El sistema no permite generar certificados para ciertos usuarios o actividades.<br>- El administrador no puede editar los certificados ya generados. | Media |
